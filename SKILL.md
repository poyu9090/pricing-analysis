# 分市場訂閱定價分析

## 背景與目的

此 skill 幫 PM 系統性地評估 BeautyPlus、BeautyCam、Wink、Meitu 在各市場的訂閱定價是否合理，並提出具體調整建議。分析從三個角度切入：
- **競品定位**：以 MAU 加權四分位數反映市場定價分布（含/不含 Capcut、Inshot、Canva 兩版）
- **市場背景**：GDP / PPP 數據供購買力認識，不做換算
- **方案折扣率**：各訂閱週期的折扣比例供參考

---

## Step 0：收集本次分析所需資訊

每次執行時，詢問使用者以下資訊（可一次問清楚，不要分多輪）：

1. **產品**：BeautyPlus / BeautyCam / Wink / Meitu
2. **市場**：國家名稱與當地貨幣
3. **競品資料**：圖片或文字表格，需包含：競品名稱、訂閱方案定價（年/季/月，可能只有部分方案）、MAU；自家產品也需在表格內

若使用者一次提供多個市場（如 Excel），讓使用者確認每個工作表對應的市場，再逐一分析、最後彙整跨市場比較表。

---

## Step 1：解析競品資料

從使用者提供的圖片或表格提取：
- 競品名稱、各方案價格、MAU
- 標記哪一筆是「自家產品」

若自家產品在表格中不明顯（例如品牌名有多種變體），詢問使用者確認，不要猜測——標記錯誤會影響整份報告的核心結論。

**資料解析完成後，詢問週期比較選擇：**

根據資料中出現的方案，列出本次市場有哪些訂閱週期（年 / 月 / 週），然後問：

> 本次資料包含 [年/月/週] 方案。折扣分析要比較哪兩個週期？（例如：年 vs 月、年 vs 週）

- 若只有一個週期，Step 3c 直接跳過。
- 若使用者未回答，預設使用「年 vs 月」（有月方案時）或「年 vs 週」（無月方案但有週方案時）。
- 使用者的選擇決定 Step 3c 的計算公式與納入競品範圍。

## Step 2：抓取 GDP 與匯率資料

**先讀本地快取**：查看 `~/Desktop/Claude/pricing-data/market-reference.json`，若已有該市場的 `nominal_gdp_usd`、`ppp_gdp_usd`、`bigmac_usd`，直接使用，**不需重新搜尋**。

**若市場不在快取中**，執行三個搜尋（並行）：

```
WebSearch: "[country] GDP per capita PPP current international dollars World Bank OR IMF 2023 OR 2024"
WebSearch: "United States GDP per capita PPP current international dollars 2023 OR 2024"
WebSearch: "[country] Big Mac Index price USD 2024 OR 2025 site:economist.com OR bigmacindex.org"
```

搜尋完成後，將新市場資料補充到 `market-reference.json` 的 `markets` 物件中，格式參考現有欄位。

**匯率一律即時查詢**（時效敏感，不快取）：

```
WebSearch: "1 USD to [local currency] exchange rate today"
```

整理後，**先呈現給使用者確認再繼續**：


使用者確認後才進行分析。這步驟確保分析基準正確，尤其當 GDP 數據有多年版本時，使用者可以選擇最貼近決策情境的數字。

## Step 3：三維度分析

### 3a. 競品定價全覽（年訂閱為主）

MAU 是月活躍用戶，包含免費用戶，不代表付費用戶數。MAU 在此作為「市場能見度」權重——MAU 高的 app 對用戶定價感知影響更大，因此四分位數計算以 MAU 加權。

Capcut 在多數市場 MAU 遠超其他 app，且屬影片剪輯品類。**每市場輸出兩版分析**：含 Capcut（廣義市場）和不含 Capcut（美顏/相機直接競品）。

**四分位數分析**（兩版分別計算，每版各輸出兩組）

將競品依年訂閱價由低到高排序，分別以兩種方法計算：

**1. 簡單四分位數**（每款 app 等權重）
```
Q1（25th）= 價格排序第 25 百分位
Q2（50th）= 中位數
Q3（75th）= 價格排序第 75 百分位
```

**2. MAU 加權四分位數**（以 MAU 為權重計算累積分布）
```
Q1（25th）= 累積 MAU 達 25% 時的價格點
Q2（50th）= 累積 MAU 達 50% 時的價格點
Q3（75th）= 累積 MAU 達 75% 時的價格點
```

兩組均標出自家產品落在哪個區間：
- 低於 Q1：< Q1
- Q1 ~ Q2：Q1~Q2
- Q2 ~ Q3：Q2~Q3
- 高於 Q3：> Q3

所有本地貨幣價格同步換算為 USD（供跨市場比較）。

### 3b. GDP / PPP 市場認識

展示以下數據供市場背景參考，不做定價換算：

| 項目 | 數值 | 來源 |
|------|------|------|
| [市場] GDP per capita (PPP) | $XX,XXX | 世界銀行 XXXX 年 |
| 美國 GDP per capita (PPP)   | $XX,XXX | 世界銀行 XXXX 年 |
| GDP 比率（市場 / 美國）      | XX%     | — |
| 匯率                        | 1 USD = X.XX [貨幣] | [來源] |

### 3c. 訂閱週期省多少（依 Step 1 選定的週期執行）

**只納入同時有兩個比較週期的競品**，計算「選長方案比短方案便宜多少」：

| 比較組合 | 公式 |
|---------|------|
| 年 vs 月 | `(月訂閱 − 年訂閱 / 12) ÷ 月訂閱` |
| 年 vs 週 | `(週訂閱 − 年訂閱 / 52) ÷ 週訂閱` |
| 月 vs 週 | `(週訂閱 − 月訂閱 / 4.33) ÷ 週訂閱` |

對有資料的競品計算後，輸出：
- 各競品省多少百分比（由低到高排序）
- 簡單四分位數（Q1 / Q2 / Q3）
- 競品均值
- 自家產品落點與定位說明

## Step 4：定價建議

### 定價決策評分模型

**先執行評分，再填入建議表格。**

| 模塊 | 衡量問題 | 權重 |
|------|---------|------|
| M1 競品定位缺口 | 自家產品在市場四分位（簡單 + MAU 加權兩版均值）中偏便宜還是偏貴？ | 40% |
| M2 市場類型 | 依世界銀行人均名目 GDP 分級，高所得市場定價空間大 | 25% |
| M4 MAU 定價一致性 | MAU 低於自家產品的競品中，定價更高的比例 | 15% |
| M5 美圖旗下關係 | 自家產品與同市場 Meitu app 的定價差距 | 20% |

**各模塊評分規則（0–100，越高 = 越需要調漲）：**

**M1** = 簡單四分位分數
- < Q1 → 100 / Q1~Q2 → 70 / Q2~Q3 → 40 / > Q3 → 10

**M2**：依世界銀行名目 GDP per capita（固定門檻）：
- 高所得（> $13,845）→ 100
- 中高所得（$4,466–$13,845）→ 60
- 中低所得（$1,146–$4,466）→ 20（同時觸發 Hard Gate）
- 低所得（< $1,146）→ 0（觸發 Hard Gate）

**M4** = (MAU 低於自家的競品中，定價高於自家的比例) × 100

**M5** = 自家 vs 同市場 Meitu app 定價差距：
- 差距 > 20% → 100 / 10~20% → 80 / 5~10% → 60 / < 5% → 30
- 自家 > Meitu app → 10（可能偏高）
- 無 Meitu app 資料 → 50（中性）

**綜合分數 = M1×0.40 + M2×0.25 + M4×0.15 + M5×0.20**

| 分數 | 建議 |
|------|------|
| ≥ 70 | 調漲 |
| 40~69 | 維持 |
| < 40 | 調降 |

**Hard Gate（任一條件觸發即覆蓋分數）：**
- 市場為 LMIC / LIC → 強制維持
- 調整後折扣偏移 > 25pp → 提示需同步調整其他方案

> M6 收入貢獻：資料到位後加入，目前停用。加入時按比例重分配權重。

---

針對每個訂閱方案（年/季/月）輸出：

| 欄位 | 說明 |
|------|------|
| 建議 | 調漲 / 維持 / 調降 |
| 調整幅度 | 顯示實際幅度至小數點第二位（如 +11.77%），不強制取整至 5% 倍數 |
| 建議新價 | 本地貨幣，參考市場定價習慣取整（台灣 NT$10 整、日本 ¥100 整，避免 NT$1,237 這類不自然的數字） |
| USD 等值 | 按即時匯率換算 |
| 調整後定位 | 調整後落在哪個四分位區間（含 Capcut / 不含 Capcut 兩版） |

附 3-5 句理由，說明建議主要依據哪個維度（競品位置 / GDP 理論區間 / MAU 定位一致性）。

若多個方案同時調整，計算調整後的折扣率偏移，若超過 25 個百分點則提示需同步微調其他方案以維持比例。

### 定價建議卡片結構（HTML 報告）

定價建議以**獨立卡片**呈現，位置在 header 正下方、圖表之前，包含以下三層：

**① rec-box（綠色建議框）**
- 標題：建議：[年/月] 訂閱調漲/維持/調降 +XX.XX%
- 三格：現行價 / 建議新價（標紅） / 月訂閱（如有）
- 建議理由：1–2 句說明主要依據

**② 市場價格四分位（簡單，4 格橫排）**
- 顯示 4 個區間：低於 Q1 / Q1~Q2 / Q2~Q3 / 高於 Q3
- 每格顯示對應價格區間
- 自家產品調前位置（粉色標籤）標在對應格，調後位置（綠色標籤）標在對應格

**③ 方案明細表**
- 欄位：方案 | 建議 | 幅度 | 建議新價 | USD 等值 | 折扣率變化（調前→調後） | 四分位（簡單） | 四分位（MAU 加權）
- 若有同集團其他產品，在明細表下方附「集團旗下產品年月價格對比」子表（含：年訂閱 / 月訂閱 / MAU / 年比月省）

**④ 定價決策評分卡片（在定價建議卡片正下方獨立呈現）**
- 4 格模塊說明（每格含指標描述 + 評分規則）
- 評分明細表（模塊 / 權重 / 市場指標數據 / 得分 / 加權分）
- 綜合分數進度條（0–100，標示 40 / 70 兩個門檻）+ 決策結論 badge

## Step 5：輸出報告

6. **折扣率長條圖**（年 vs 月），自訂 canvas plugin 繪製均值虛線 + 自家/集團標籤
7. **GDP 雙欄圖**（左：人均名目 GDP 橫條圖；右：Big Mac Index 橫條圖），`grid-2` 並排
8. **完整競品定價表**（含無年方案的 Snapseed / Retake 等 app）

顏色規則：`own`（自家分析品）= `#E91E8C`（粉）；`meitu`（集團旗下）= `#FF6B35`（橘）；`capcut` = `rgba(92,107,192)`（藍）；`other` = `rgba(144,164,174)`（灰）。

rec-box 與最終得分面板顏色依建議方向：調漲 = 綠（`#A5D6A7`/`#66BB6A`）；維持 = 琥珀（`#FFB300`/`#FFFDE7`）；調降 = 紅（`#EF9A9A`/`#EF5350`）。

**不可使用** `chartjs-plugin-annotation`，Q 線改用自訂 canvas inline plugin 繪製（同 BeautyCam_UK 做法）。

- 跨市場 USD 比較表（彩色標記建議方向）
